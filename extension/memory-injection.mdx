---
title: "Memory Injection"
description: "Two ways to inject memory into AI prompts: Improve Prompt and Add Context."
---

Memory injection puts your stored knowledge into active conversations. Two modes: **Improve Prompt** rewrites your draft using relevant memories, and **Add Context** prepends selected memories to your prompt.

## Improve Prompt

Write a rough prompt, then trigger improvement via the overlay or `Cmd+Shift+I`. Memorybase searches your memories, rewrites your draft, and shows a preview with the original, improved version, context used, and token count. Accept, edit, or cancel.

Three improvement strategies (configurable per profile):

| Strategy | Effect |
|----------|--------|
| **Expand** | Adds detail and specificity |
| **Refine** | Cleans up language and ambiguity |
| **Format** | Adds structure (headers, bullets) without changing content |

## Add Context

Click the overlay button, browse your memory library, and select memories to attach. A token budget tracker keeps you within limits (default: 2,000 tokens). Selected memories are prepended as a structured context block:

```xml
<context source="memorybase">
[Memory 1 content]
---
[Memory 2 content]
</context>

[Your original prompt]
```

## Settings

| Setting | Default | Description |
|---------|---------|-------------|
| Token budget | 2,000 | Max tokens across all attached memories |
| Similarity threshold | 60% | Minimum match score for surfacing memories |

## Fallback

If the adapter's prompt injection is broken, context is copied to your clipboard instead. Paste manually. All other features continue working.
