---
title: "Profiles: keep work and personal memory separate"
description: "Profiles are isolated memory compartments in Memorybase. Each has its own memories, capture settings, and search index — data never bleeds between profiles."
---

A profile is a self-contained memory compartment. Every memory you capture, every conversation you save, and every search you run belongs to exactly one profile. Profiles never share data with each other unless you explicitly take action — switching profiles gives you a clean slate for a completely different context.

The most common use case is keeping work and personal memories separate. But profiles go further than that: you can isolate individual client projects, maintain separate memory for different programming languages, or keep creative writing separate from technical research.

## The General profile

Memorybase ships with a **General** profile that is always present and cannot be deleted. When you first install the extension, all captures go to General. You can rename it and configure it like any other profile, but it must always exist as the fallback.

## Creating a profile

<Steps>
  <Step title="Open the extension popup">
    Click the Memorybase icon in your Chrome toolbar.
  </Step>
  <Step title="Navigate to profile management">
    Click **My Memory** → **Manage** to open the memory library in the dashboard, where you can create and manage profiles.
  </Step>
  <Step title="Create the profile">
    Click **New Profile**, enter a name (up to 50 characters), and confirm. The profile is immediately available for selection.
  </Step>
</Steps>

<Tip>
  Use descriptive names like "Work – Client A", "Personal", or "Creative Writing" so the active profile indicator in the overlay always makes the context clear at a glance.
</Tip>

## Switching profiles

You can switch your active profile in two ways:

- **Popup dropdown** — open the extension popup and select a different profile from the dropdown in the memory section.
- **Quick-switcher** — press `Ctrl+Shift+P` (or `Cmd+Shift+P` on Mac) to open the quick-switcher modal. Use the arrow keys to navigate and Enter to confirm.

When you switch profiles, the extension immediately begins capturing to the new profile and all memory searches query only that profile's data.

## Profile settings

Each profile has its own settings that control how it captures, improves, and injects memory.

<AccordionGroup>
  <Accordion title="Auto-save">
    **Auto-save enabled** (default: on) — when enabled, Memorybase automatically captures new conversations on supported sites. Disable this if you want to save content manually for this profile only, without affecting auto-save on other profiles.

    **Minimum message length** (default: 20 characters) — messages shorter than this threshold are not captured. Increase it to skip short, low-value exchanges like "thanks" or "ok".
  </Accordion>
  <Accordion title="Improvement style">
    Controls how **Improve Prompt** rewrites your drafts for this profile.

    | Style | Behavior |
    |-------|----------|
    | **Expand** | Adds detail, context, and specificity |
    | **Refine** | Cleans up language and fixes ambiguity |
    | **Format** | Adds structure without changing content |

    A coding profile might use Expand to add technical constraints. A writing profile might use Refine to tighten language.
  </Accordion>
  <Accordion title="Injection token budget">
    **Default: 2,000 tokens.** Sets the maximum total tokens that can be injected when you use Add Context. Increase this if you frequently work with large context blocks and your AI platform supports a long context window. Decrease it if you find injections make prompts unwieldy.
  </Accordion>
  <Accordion title="Minimum similarity threshold">
    **Default: 60%.** Controls which memories appear in the Add Context panel. Only memories that match your current context above this threshold are shown. Raise it to see fewer, higher-relevance results. Lower it to surface more memories, including loosely related ones.
  </Accordion>
  <Accordion title="Retention">
    **Default: forever.** You can set a retention period for memories in this profile: 30, 90, 180, or 365 days. Memories older than the retention period are automatically deleted during cleanup. Set a retention period if you work on time-limited projects and want stale context to clear itself out.
  </Accordion>
</AccordionGroup>

## Profile statistics

Each profile shows:

- **Memory count** — total number of memories stored
- **Storage used** — bytes consumed by this profile's memories and embeddings
- **Last activity** — when the most recent memory was captured or accessed

You can see these stats in the memory library on the dashboard.

## Data isolation

Searches only query the active profile. When you run a search or trigger Add Context, Memorybase never looks at memories from other profiles. The active profile name is always visible in the overlay button and popup so you always know which context is in use.

<Warning>
  There is no cross-profile search by default. If you need to find something across all profiles, you can enable the "Search All Profiles" toggle in the memory library — but this is an explicit action, not the default behavior.
</Warning>

## Deleting a profile

Deleting a profile permanently removes all memories associated with it. This action cannot be undone.

<Steps>
  <Step title="Open profile settings">
    Go to the memory library in the dashboard and open the profile you want to delete.
  </Step>
  <Step title="Click Delete profile">
    Scroll to the bottom of the profile settings and click **Delete profile**.
  </Step>
  <Step title="Confirm deletion">
    A confirmation dialog appears listing how many memories will be deleted. Type the profile name to confirm, then click **Delete permanently**.
  </Step>
</Steps>

<Warning>
  Deleting a profile removes all its memories immediately and permanently. Export your data first if you want a backup. You cannot delete the General profile.
</Warning>

## Common use cases

<AccordionGroup>
  <Accordion title="Separating work clients">
    Create one profile per client. Each client's terminology, coding standards, and project constraints stay isolated. Switch profiles with `Cmd+Shift+P` between client sessions. Context from Client A never appears in Client B's suggestions.
  </Accordion>
  <Accordion title="Personal vs. professional">
    Keep a Personal profile for hobbies, learning, and creative projects. Keep a Work profile for professional tasks. Use different retention settings — longer for professional knowledge, shorter for time-sensitive personal research.
  </Accordion>
  <Accordion title="Different languages">
    If you work in multiple spoken languages, separate profiles prevent cross-language context bleed. Set the improvement style per profile to match writing conventions for each language.
  </Accordion>
  <Accordion title="Time-limited projects">
    Set a retention period on a project profile so memories expire automatically after the project ends. This keeps your memory library clean without manual cleanup.
  </Accordion>
</AccordionGroup>
