title: "Projects"
description: "Group related conversations and sources into named workspaces that generate context packs."
---

A project groups related conversations and custom sources around a theme — a client, a research area, a codebase. Memorybase processes the conversations and produces a [context pack](/features/context-packs) you can inject into any AI session.

## Creating a project

Open the **Projects** tab, click **New project**, give it a name (up to 200 chars) and an optional description (up to 1,000 chars). The description is included in your context pack, so make it useful.

## Adding conversations

From the project: click **Add conversations** and select from the list (up to 100 at once). From a conversation: click **Add to project**. Adding conversations queues them for context pack processing.

Remove a conversation by clicking **Remove** next to it — this unlinks it but does not delete it.

## Sources

Attach reference material that isn't a conversation:

<CardGroup cols={3}>
  <Card title="note" icon="note-sticky">Freehand text — decisions, requirements, background context.</Card>
  <Card title="url" icon="link">Links to docs, PRs, design files, or any external resource.</Card>
  <Card title="snippet" icon="code">Code blocks — key functions, schemas, configs.</Card>
</CardGroup>

Click **Add source** in the project, set the type and content, and it's attached immediately.

## AI suggestions

Memorybase analyzes your recent conversations and suggests new projects. Open the **Projects** tab and click **Suggestions** to review and create them with conversations pre-linked.

## Archiving

Archive a project to hide it from the main list. Conversations, sources, and the context pack are all preserved. Archiving a project does not archive its conversations.

## System projects

**Uncategorized** shows conversations not in any project. **Archived** shows all archived conversations. Both are system-managed.
