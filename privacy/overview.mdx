Your data stays on your device by default. All processing — including vector embeddings — happens locally.

## Where data lives

| Data type | Stored | Leaves browser? |
|-----------|--------|-----------------|
| Memories | Local storage | Never (unless exported) |
| Embeddings | Local storage | Never |
| Settings | Local storage | Only with Chrome Sync |
| Embedding model | Local cache | Downloaded once from CDN |

## On-device PII sanitization

Memorybase scrubs PII from conversations on-device before any processing or cloud sync. Raw text is never transmitted externally.

## Encryption

**Local storage** — Optional. Set a password in **Settings > Security** to enable AES-256 encryption. Without it, data relies on Chrome's storage isolation.

**Export files** — Always encrypted with a password you set at export time. No recovery if lost.

**Data in transit** — HTTPS only. The only outbound request is the one-time embedding model download.

## Chrome permissions

- **storage** — Saves settings and whitelist locally.
- **activeTab** — Reads the current tab only. No background access.
- **scripting** — Injects the toolbar into AI chat pages.
- **tabs (optional)** — Multi-tab awareness. Off by default.

**Host permissions** — Limited to `chatgpt.com`, `chat.openai.com`, and `claude.ai`. Other sites require explicit whitelist addition.

## Privacy controls

- **Site whitelist** — Only captures on sites you enable. None by default.
- **Pause capture** — One-click toggle in the toolbar.
- **Delete memories** — Immediate and permanent. No trash.
- **Export and wipe** — Full encrypted backup, then wipe. See [Export & wipe](/privacy/data-export).
- **Telemetry** — Off by default. Opt-in sends anonymous stats only, never content.

## Storage limits

| Threshold | Behavior |
|-----------|----------|
| 80% full | Warning shown, auto-save continues |
| 95% full | Auto-save pauses until you free space |

Configure in **Settings > Storage**: max memories per profile (100-5,000), retention period (30/90/180/365 days or forever), and cleanup strategy (oldest, least-accessed, or lowest-relevance first).
