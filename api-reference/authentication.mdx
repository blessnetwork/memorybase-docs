---
title: "Authentication"
description: "Obtain and use bearer tokens for the Memorybase API via email OTP."
---

Every request requires a bearer token obtained through the email OTP flow.

## Get a token

1. Open the Memorybase extension, enter your email.
2. Enter the one-time passcode sent to your inbox.
3. The extension stores the session token automatically.

For direct API access, retrieve the token from the extension's storage or the session cookie after OTP verification.

## Use the token

```bash
Authorization: Bearer <your-token>
```

```bash
curl https://api.memorybase.app/settings/profile \
  -H "Authorization: Bearer <your-token>"
```

## Token expiry

Sessions persist while active. Expired or revoked tokens return `401`. Re-authenticate via OTP to get a new token. There is no programmatic refresh endpoint.

<Warning>
  Treat your token like a password. Do not commit it to version control or expose it in client-side code.
</Warning>
