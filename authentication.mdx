---
title: "Authentication: Passwordless email sign-in"
description: "Memorybase uses one-time email codes to sign you in — no password to create or remember. Sessions persist across browser restarts automatically."
---

Memorybase uses passwordless authentication. When you sign in, you enter your email address and receive a 6-digit one-time code. Enter the code to verify your identity, and the extension links to your account. There are no passwords to create or manage.

## How to sign in

<Steps>
  <Step title="Open the extension popup">
    Click the Memorybase icon in your Chrome toolbar. If you're not signed in, the onboarding screen appears with a **Sign In** button.
  </Step>

  <Step title="Click Sign In">
    Click **Sign In**. Memorybase opens a sign-in page in a new browser tab.
  </Step>

  <Step title="Enter your email address">
    Type your email address into the field and click **Continue**. Memorybase sends a 6-digit code to that address. The code expires in 10 minutes.
  </Step>

  <Step title="Enter the one-time code">
    Check your inbox for an email from Memorybase. Enter the 6-digit code on the sign-in page and click **Sign in**.

    Once verified, the page shows a confirmation message. You can close the tab — the extension updates automatically.
  </Step>
</Steps>

## How sessions work

After signing in, your session persists across browser restarts. You do not need to sign in again when you reopen Chrome or your computer. The extension checks your authentication status in the background each time the popup opens.

If you sign out manually (available in the extension settings), you'll need to go through the email OTP flow again to resume capturing conversations.

## Troubleshooting

<AccordionGroup>
  <Accordion title="The sign-in code didn't arrive">
    Check your spam or junk mail folder first — transactional email from new senders is often filtered.

    If the email isn't there after a couple of minutes, go back to the sign-in page and click **← Use a different email** to restart the flow with the same or a different address.

    <Note>
      Each new code request invalidates any previously sent code. If you requested multiple codes, only the most recent one is valid.
    </Note>
  </Accordion>

  <Accordion title="The sign-in code expired">
    Codes expire after 10 minutes. If yours has expired, click **← Use a different email** on the code entry screen to return to the email step, then request a new code.
  </Accordion>

  <Accordion title="The extension popup still shows the sign-in screen after I verified">
    The popup listens for a message from the sign-in page tab to update automatically. If the popup was closed before verification completed, reopen it — it checks your auth status each time it loads.

    If the popup still shows the sign-in screen, try clicking **Sign In** again. If the sign-in page shows "Connected!", your session is active and the popup should reflect that on the next open.
  </Accordion>

  <Accordion title="I signed in on one device — do I need to sign in again on another?">
    Yes. Each browser instance requires its own sign-in. Go through the email OTP flow on each device where you install the extension. Once signed in, your memory syncs automatically across all devices.
  </Accordion>

  <Accordion title="I want to sign out or switch accounts">
    Open the extension popup and navigate to **Settings**. From there you can sign out of your current session. To sign in with a different email address, sign out first, then use the **Sign In** button on the onboarding screen.
  </Accordion>
</AccordionGroup>

<Warning>
  If you lose access to your email address, you will not be able to sign in to your Memorybase account. Make sure the email you use is one you control long-term.
</Warning>
